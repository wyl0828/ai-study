Two Sum

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出和为目标值 target 的那两个整数，并返回它们的数组下标。

假设每种输入只会对应一个答案，且同一个元素不能使用两遍。

示例 1：

输入：nums = [2,7,11,15], target = 9
输出：[0, 1]
解释：nums[0] + nums[1] == 9

示例 2：

输入：nums = [3,2,4], target = 6
输出：[1, 2]

约束条件：

  • 2 <= nums.length <= 10^4
  • -10^9 <= nums[i] <= 10^9
  • -10^9 <= target <= 10^9
  • 只会存在一个有效答案

psychology错误诊断

你的解法使用了暴力枚举，时间复杂度为 O(n²)。虽然逻辑上能覆盖部分用例，但在处理 nums = [3,2,4], target = 6 时，由于双重循环的起始条件，导致 i 和 j 不能正确配对相同索引附近的元素。

tips_and_updates改进建议

考虑使用哈希表将查找时间从 O(n) 降到 O(1)。遍历数组时，对于每个元素，先检查 target - nums[i] 是否已在哈希表中，再将当前元素存入。

2 知识点提示

关键知识点是 HashMap。遍历数组时，用 HashMap 存储已遍历过的元素值和下标。对于当前元素 nums[i]，检查 target - nums[i] 是否作为 key 存在于 HashMap 中。
