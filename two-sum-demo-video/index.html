Bug 提交 · WRONG_ANSWER
101-two-sum-bug.java ACM / Main 
Map<Integer, Integer> map = new HashMap<>();
for (int i = 0; i < n; i++) {
  map.put(nums[i], i);
  int need = target - nums[i];
  if (map.containsKey(need)) {
    System.out.println(map.get(need) + " " + i);
    return;
  }
}
失败用例观察 WRONG_ANSWER

输入

n = 2

nums = [3, 3]

target = 6

期望

0 1

实际

0 0

问题核心

当前元素先写入 HashMap，导致它可以立刻和自己匹配。重复元素场景下，下标组合被判错。

AI 诊断 · Observation → Error Classification

错误类型

LOGIC_ERROR HashMap 基础查找

错误原因

代码在检查 complement 之前插入当前值，造成 self-matching。

改进建议

遍历时先查 target - nums[i] 是否已出现，再记录当前 nums[i]。

for (int i = 0; i < n; i++) {

  if (map.containsKey(need)) {
    System.out.println(map.get(need) + " " + i);
